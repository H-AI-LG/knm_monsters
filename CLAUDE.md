## 세계관 — "전생의 도사, 다시 깨어나다"

### 핵심 설정
- **주인공** = 시대를 넘나들며 정령과 요괴를 다스리던 도사. 환생하며 모든 기억을 잃었다가 박물관에서 각성
- **도슨트 정령** = 전생에 도사님을 동행하던 작은 정령. 환생을 못 하고 박물관에서 기다리던 존재 (표지 흰 마스코트)
- **유물 정령** = 잠들어 있던 수호 정령. 퀴즈로 진짜 도사인지 확인 후 수호신으로 각성
- **경천사탑** = 도사가 전생에 직접 봉인한 불국토의 핵. 최종 보스가 아니라 봉인 강화 엔딩

### 도입 대사 (도슨트 정령)
> "잘 왔네, 도사 양반. 자네는 본래 이 땅의 혼란을 잡던 도사였네. 유물에 깃든 정령들을 다스리고, 시대마다 자네의 부적과 도력으로 균형을 지켰지. 하지만 시간이 흘러 자네는 환생하며 모든 걸 잊었고, 정령들은 잠들었네. 이제 자네가 돌아왔으니, 다시 정령들을 깨우고 균형을 잡아야 한다네."

### 배틀 흐름
```
시간 균열 → 정령 등장
  → "...이 기운... 도사님이신가? 다시 오셨군요."
  → 대화 (정령이 자신의 이야기를 나눔)
  → 시험 (퀴즈) — "진짜 도사인지 확인하겠습니다"
  → 정답: 도력(道力) 공격 → 봉인력(封印力) 0 → "정령이 깨어납니다"
  → 오답: 봉인이 버팀 → 도력 감소 → 재도전
  → "다시 제 정령은 도사님의 수호신이 되겠습니다" → 수호부(守護符) 등록
```

### HP 시스템
| 항목 | 이름 | 설명 |
|---|---|---|
| 플레이어 HP | **도력(道力)** | 퀴즈 오답 시 감소. 0되면 배틀 종료(재도전 가능) |
| 유물 HP | **봉인력(封印力)** | 퀴즈 정답 시 감소. 0되면 정령 각성·획득 |

### 도감 명칭
- 도감 → **수호부(守護符)** 또는 **전생부**
- 수호신 획득 → **정령 각성**
- 도감 등록 → **수호부에 각인됨**

### 엔딩 (30선 완료)
전생부 완성 → 빛 폭발 → 모든 정령이 인사 → AI 회상 내레이션 (형준 파트):
> "자네의 전생은 시대를 넘나들던 도사였네. 구석기엔 주먹도끼를, 고려엔 청자 장인에게 도력을 빌려주었으며, 조선엔 사관들의 붓에 정기를 불어넣었지. 오늘 자네는 모든 정령을 다시 깨우고, 박물관의 균형을 회복했네."

### 확장 (공모전 이후)
- **사신 4기** = 30선 완료 후 보너스 라인 (청룡/백호/주작/현무)
- **관심사 조사** = 2~3문항으로 전생도 생성, AI가 맞춤 정령 순서·페르소나 결정
- **요괴 라인** = 잠든 정령(우호)과 별개의 적대 미니배틀

## 기술 스택
- **프론트엔드**: React 19 + Phaser 3 + Vite
- **모바일 지원**: 조이스틱 UI (PhaserGame.jsx)
- **맵 시스템**: 타일맵 + 픽셀 일러스트 배경 (실제 박물관 1층 도면 기반)
- **유물 수호몹 생성**: Gemini Nano Banana (Google) — 픽셀아트 스프라이트

```
중앙홀(lobby) — 경천사 십층석탑이 중앙에 배치
├─ 위쪽 (중근세관)
│   ├─ 대한제국실
│   ├─ 조선실

```

## 채민 작업 범위 (이번 단계)
**목표**: 목업 데이터로 데모 굴러가는 상태까지. AI 연동은 형준이가 따로.

### 흐름 변경
- 기존: 타일 밟으면 → 바로 모달 열림
- **변경**: 타일 밟으면 → 감지 알림 → A버튼 → 모달 열림

### 1단계: 유물 감지 알림 UI
캐릭터가 유물 타일에 들어오면 화면 하단에 알림 표시:
```
┌─────────────────────────────┐
│ 🏺 유물이 감지되었습니다!    │
│ [A] 상호작용하기             │
└─────────────────────────────┘
```
- 유물 타일에서 벗어나면 알림 사라짐
- A버튼 (키보드 A키 + 모바일 화면 버튼) 누르면 모달 오픈

### 2단계: 목업 데이터 구조
`src/data/artifacts.js` 신규 생성.

**데이터 소스**: 문화체육관광부 국립중앙박물관 큐레이터 추천 소장품 정보 CSV (공공데이터)

**구조 예시**:
```js
export const ARTIFACTS = {
  "주먹도끼": {
    id: "artifact_001",
    name: "주먹도끼",
    era: "구석기시대",
    persona: "도끼 할배",
    greeting: "으랏차! 나는 100만 년 전부터 여기 있었다고! 뭐가 궁금하냐?",
    description: "구석기 시대 최고 히트 상품. 하나로 뭐든 다 하는 맥가이버 칼이야.",
    image: "/artifacts/주먹도끼.png",
    quiz: {
      question: "주먹도끼는 약 몇 년 전부터 사용됐을까요?",
      options: ["1만 년", "10만 년", "100만 년", "1000만 년"],
      answer: 2
    }
  },
  // ... 다른 유물
}
```

### 3단계: App.jsx 상태 관리
```js
const [nearArtifact, setNearArtifact] = useState(null)    // 감지만 된 상태
const [activeArtifact, setActiveArtifact] = useState(null) // A버튼 눌러서 대화 시작
```
- `nearArtifact` → 하단 알림 표시
- `activeArtifact` → 모달 열기
- 유물 타일 벗어나면 → `nearArtifact = null`, 알림 사라짐

### 데모 완성 기준
```
유물 타일 밟기
→ 하단 알림 뜸
→ A버튼
→ 모달 열림 + 유물몬 첫 인사 (persona·greeting)
→ 선택지 대화 2번
→ 퀴즈
→ 정답
→ "수호신 획득!" 연출
```

### 인계 포인트
- **AI 연동** (형준): 목업 굴러간 후 `/api/chat` 호출로 greeting/대화 부분 교체. **구조 자체는 안 바뀜.**
- **백엔드·소리** (수림): 데이터 구조 정해지면 백엔드로 옮기고 효과음 연결.

채민은 **구조만 명확히** 잡아두면 나머지는 팀원이 갈아끼움. 깔끔하게 인터페이스 설계하는 게 중요.

## 수호몹 생성 파이프라인 (Gemini Nano Banana)

### 원칙
1. **앵커 시스템**: 등급마다 화풍 기준 앵커 사용
   - 일반 = 일반_26_달항아리
   - 고급 = 고급_07_가야갑옷영체
   - 전설/보스 = 전설_22_반가사유상
2. **유물 사진 첨부** (직접 촬영 또는 박물관 공공누리 1유형)
3. **PIXEL ART, 16-bit retro RPG sprite** 명시
4. **얼굴 = 슬릿 빛 눈만** (사람 얼굴 X)
5. **CRITICAL FRAMING**: 사방 여백 + 잘림 금지
6. **종이류 유물**: "ABSOLUTE RULE — 원본 그대로 유지" 박고 정령은 분리 추가

### 확립된 패턴
- **차별화**: 매체+구조+톤+문화권으로 갈라치기 (불교 계열 = 철불·감산사·간다라·변상도 각자 다른 표현)
- **전설/보스 압도감**: 거대 스케일 + 정교 디테일 + 다중 후광 + 시그니처 동반자 + 신화 서사
- **나노바나나 한계**: 자세 지시 약함, 공간구도 어려움, 사람얼굴 박으면 디즈니화 → 첫줄에 핵심 박기
